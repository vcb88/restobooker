# План развития проекта (Next Steps)

## Этап 1: Инфраструктура и Хранение
- [ ] **1.1. Персистентное хранение данных:** Перейти с in-memory хранилища на SQLite (для простоты MVP) или PostgreSQL. Это предотвратит потерю бронирований при перезагрузке бота.
- [ ] **1.2. Модель нескольких столиков/зон:** Обновить схему данных, чтобы поддерживать не один виртуальный столик, а несколько (зал, веранда) с разной вместимостью.

## Этап 2: Расширение логики бронирования
- [ ] **2.1. Отмена и изменение брони:** Добавить в `llm_service.py` и `database.py` возможность находить существующую бронь по номеру телефона/имени и отменять её или переносить на другое время.
- [ ] **2.2. Учет количества персон:** Добавить извлечение количества гостей из запроса и проверку, влезет ли такая компания за свободные столики.

## Этап 3: Интеграция и уведомления
- [ ] **3.1. Уведомления для персонала:** Реализовать отправку сообщения администратору при создании нового бронирования.
- [ ] **3.2. Интеграция с RMS (iiko/R-Keeper):** Создать слой абстракции для работы с внешними API ресторанных систем.

## Этап 4: Полировка и Надежность
- [ ] **4.1. Логирование и мониторинг:** Улучшить сбор логов для анализа ошибок LLM.
- [ ] **4.2. Обработка конфликтов:** Более умное предложение альтернатив, если нужный столик занят.
